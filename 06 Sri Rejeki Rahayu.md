1. Native App
Point (Titik Utama): Aplikasi Native adalah aplikasi yang dibangun khusus untuk platform tertentu (seperti Android atau iOS) menggunakan bahasa pemrograman spesifik.
Reason (Alasan): Karena dibuat khusus untuk sistem operasi tertentu, aplikasi ini memiliki akses penuh ke perangkat keras (hardware) dan memberikan performa paling maksimal.
Example (Contoh): Sesuai catatan di papan: menggunakan Kotlin (Android), akses fitur NFC, BT (Bluetooth), Cam (Kamera) secara mendalam, serta sangat cocok untuk Game dan VR/AR yang butuh grafis berat.
Point (Penegasan): Jika Anda mengutamakan "High Performance" dan fitur hardware yang kompleks, Native adalah pilihan terbaik.

2. Hybrid App
Point (Titik Utama): Aplikasi Hybrid adalah kombinasi antara aplikasi web dan native yang dibungkus dalam "wadah" native.
Reason (Alasan): Pengembang hanya perlu menulis satu kode dasar (biasanya HTML/JS/CSS) yang bisa berjalan di banyak platform, sehingga lebih hemat waktu dan biaya dibanding native.
Example (Contoh): Di papan tertulis "Medium" performa, menggunakan Plugin untuk mengakses fitur HP, dan cocok untuk aplikasi tipe "App Bisnis" yang tidak terlalu menuntut grafis tinggi.
Point (Penegasan): Hybrid adalah solusi jalan tengah untuk efisiensi waktu pengembangan dengan akses fitur perangkat yang lumayan.

3. PWA (Progressive Web App)
Point (Titik Utama): PWA pada dasarnya adalah situs web yang dioptimalkan agar terlihat dan berfungsi seperti aplikasi mobile.
Reason (Alasan): PWA tidak perlu diunduh melalui App Store atau Play Store; pengguna cukup mengakses melalui URL browser. Ini sangat ringan dan mudah diperbarui.
Example (Contoh): Papan mencatat penggunaan HTML/JS/CSS + Service Worker, akses API terbatas (seperti kamera), dan distribusi utamanya melalui URL/Web App.
Point (Penegasan): PWA sangat efektif untuk menjangkau pengguna secara cepat tanpa hambatan instalasi aplikasi yang rumit.
